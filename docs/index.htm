WEB 215 - Our Members (hurry up and add yourself already)

Start by reading this:

  1. Go to the readme file for http://github.com/divonbriesen/CP_WEB215_SP2020 and make sure you've accepted the invitation to collaborate, and marked yourself as present. Update the links to your github profile and to your repo for this class.
  2. Install a github client on your computer. your teacher is using github desktop for windows, but also has used sourcetree.
  3. Clone the master repo (the one this page is in) - http://github.com/divonbriesen/CP_WEB215_SP2020 to your Computer
  4. Use your editor (notepad++ or give visual studio code a try- that's what your teacher is using and it'll help you go faster) to add your information to the index page in a similar format to the examples already there. You can tidy any/all up, but do it consistently. They should all be formatted identically though the content will be different.
  5. Save your work, look at it locally (i.e. in a browser window on your computer) and edit/save until you are happy with what you see.
  6. Commit your work in the github client, push origin in the github client, then view the page on github pages (http://divonbriesen.github.io/CP_WEB215_SP2020/ or http://wesome.us/CP_WEB2015_SP2020/). Note this format - in my case it reroutes the domain name to a name i setup, but it's the same. (i.e. divonbriesen.github.io = wesome.us)
  7. Since others will be editing on and off it's important to commit and push often, to avoid conflicts. Pull origin just before your start working, and push origin when done. If you were to make a change, then push it 8 hours later, it's likely someone else will have made a change as well and you'll get a merge conflict. Using Visual Studio Code will help because it'll let you resolve conflicts within the editor!
  8. As you become a more advanced developer, you will want to view your pages via a webserver, rather than just opening them locally in a browser. If you install xampp on your computer, you can run an Apache (the A in XAMPP) webserver - be sure to put your web files in the htdocs folder inside the xampp folder - then you can browse to the page using http://localhost/whateverfolderyoucreated/whateveryourfileiscalled - so for example, in this class, a local webserver url could be: http://localhost/CP_WEB215_SP2020/
  9. On github you can make any repository into a simple webserver (no php, but JS will work) - using settings, and the github pages block. I recommend making a docs folder inside your repo and using that- to distinguish from other files you may not want on the website.
